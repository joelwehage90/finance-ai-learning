## Insikter

- Klient: Den som ber om något. Server: Den som svarar. Tänk restaurang. Klient är kund, köket är servern. Behöver inte veta hur köket fungerar! Men - ett program kan vara både klient och server beroende på kontexten.
    - I min Finance AI Agent:
Python-verktyget hämtade data från Supabase → Python var klient, Supabase var server
Din agent anropade ditt API → Agenten var klient, ditt API var server

- stdin/stdout = standardkanaler som alla program har:
    - stdin (standard in) = dit programmet läser indata
    - stdout (standard out) = dit programmet skriver utdata
    - FastMCP sköter allt detta åt oss 

- client credentials: client id + client secret --> skickar dem och får tillbaka access token. Slipper lagra och hantera access token från OAuth

- En MCP-server är i vårat fall som ett mellanlager mellan en klient och API-server. En MCP-server "wrappar" olika API-endpoints och tillgängliggör dem för klienten på ett standardiserat sätt. Ett verktyg i en mcp-server kan dessutom innehålla många fler anrop, och bearbetning av datan. En MCP-server är självbeskrivande, vi behöver inte på förhand veta vad det finns för verktyg och hur dem funkar.
    - En MCP-server måste inte wrapa ett API. Den kan göra helt andra saker. T ex skriva SQL mot en databas, köra python-kod,
